Bioacoustic Template Labeler

Aplicación web pura para marcar una plantilla sobre un espectrograma y buscar regiones acústicas similares por embeddings locales. Todo corre en el navegador: no hay servidor, no hay Python en producción y el audio no se sube a internet.

Inicio rápido

  1. Abre la aplicación desde index.html servida por un servidor local o desde GitHub Pages.
  2. Pulsa Subir / cambiar audio o arrastra un audio en el modal inicial.
  3. La aplicación decodifica el audio y construye el espectrograma automáticamente.
  4. Arrastra una caja roja sobre el patrón acústico que quieres usar como plantilla.
  5. Pulsa Guardar plantilla.
  6. Ajusta o deja activo el autoajuste de score mínimo y separación entre ventanas.
  7. Pulsa Buscar similares.
  8. Exporta los candidatos como CSV, XLSX o TXT Audacity.
La construcción del espectrograma es automática después de subir el audio. No hay un botón separado para construirlo.

Ejecutar localmente

No abras el archivo con doble clic como file:///.... Algunos navegadores bloquean los Web Workers y la carga de módulos locales. Sirve la carpeta como sitio web local.

Con Conda o Python

cd C:\Users\Usuario\Downloads\bioacoustic-template-labeler-wizard
python -m http.server 8000

Luego abre:

http://localhost:8000

Con Node.js

npx --yes http-server . -p 8000 -c-1

Flujo de uso

El flujo está diseñado como una sola pantalla de trabajo con guía contextual:

  1. Subir audio: se carga el archivo local.
  2. Procesamiento automático: se decodifica el audio y se calcula el espectrograma.
  3. Exploración: el reproductor inferior controla el audio y el espectrograma puede seguir la reproducción.
  4. ROI: se dibuja una caja sobre el patrón acústico.
  5. Plantilla: la ROI se guarda como referencia.
  6. Búsqueda: se comparan ventanas candidatas contra la plantilla.
  7. Resultados: se dibujan cajas azules y se llena la tabla de candidatos.

    8. La tabla de resultados puede ordenarse haciendo clic en los encabezados de columna. Por defecto, se mantiene el orden de mayor score primero.

  8. Exportación: se descarga un CSV con las cajas candidatas.

Parámetros

Configuración del espectrograma

  • Escala de frecuencia: lineal o mel.
  • Color: magma o blanco y negro.

ROI plantilla

  • tmin, tmax: límites temporales en segundos.
  • fmin, fmax: límites de frecuencia en Hz.

Búsqueda por embedding

  • Método de comparación: coseno, correlación normalizada, euclidiana normalizada o correlación cruzada.
  • Score mínimo: umbral desde 0 hasta 0.99. Valores altos son más estrictos; valores bajos muestran más coincidencias, pero pueden incluir ruido.
  • Separación entre ventanas: cada cuántos segundos se prueba una nueva ventana candidata. Menor es más fino y lento; mayor es más rápido pero puede saltarse eventos.
  • Autoajuste: al buscar, estima la separación según la duración de la ROI, convierte la curva densa de scores en picos/islas temporales, excluye la plantilla original para estimar el corte y usa barandas internas para evitar tanto cientos de cajas como una sola coincidencia cuando sí existen más candidatos razonables.

Formulación matemática

Esta sección describe el problema completo que resuelve la app: a partir de una plantilla marcada por el usuario en el espectrograma, se buscan eventos acústicos similares en el mismo audio.

1. Señal y espectrograma

Sea $x[n]$ una señal de audio discreta, donde $n$ indexa muestras. La aplicación calcula una transformada de Fourier de tiempo corto:

$$X[k,m] = \sum_n x[n] w[n - mH] e^{-2\pi i k n / N}$$

donde:

  • $N$ es el tamaño de la ventana de Fourier.
  • $H$ es el salto entre ventanas, también llamado hop length.
  • $w$ es la ventana de análisis.
  • $k$ indexa la frecuencia.
  • $m$ indexa el tiempo.

Se trabaja con la magnitud logarítmica en decibeles:

$$S[k,m] = 20\log_{10}(|X[k,m]| + \varepsilon)$$

Aquí $S$ es la matriz del espectrograma y $\varepsilon > 0$ evita problemas numéricos con logaritmos de cero.

2. ROI o plantilla acústica

El usuario dibuja una región de interés, o ROI, sobre el espectrograma:

$$R = [t_{\min}, t_{\max}, f_{\min}, f_{\max}]$$

donde $t_{\min}$ y $t_{\max}$ son los límites temporales en segundos, y $f_{\min}$ y $f_{\max}$ son los límites de frecuencia en Hz.

La ROI se convierte a índices discretos del espectrograma:

$$R \rightarrow [i_{f0}, i_{f1}, i_{t0}, i_{t1}]$$

y se extrae el patch de plantilla:

$$P_R = S[i_{f0}:i_{f1}, i_{t0}:i_{t1}]$$

La duración de la plantilla es:

$$T_R = t_{\max} - t_{\min}$$

2b. Plantilla compuesta con varias muestras

Opcionalmente, una plantilla puede construirse a partir de varias muestras del mismo fonotipo marcadas por el usuario:

$$\mathcal{R} = \{R_1, R_2, \dots, R_L\}$$

Cada muestra $R_l$ tiene límites $t_{\min,l}$, $t_{\max,l}$, $f_{\min,l}$ y $f_{\max,l}$. Para que una muestra extrema no agrande artificialmente la plantilla, el soporte frecuencial se define con cuantiles:

$$f_{\min}^{*} = Q_{0.10}(f_{\min,1}, \dots, f_{\min,L})$$

$$f_{\max}^{*} = Q_{0.90}(f_{\max,1}, \dots, f_{\max,L})$$

La duración común se toma como la mayor duración observada:

$$T^{*} = \max_l(t_{\max,l} - t_{\min,l})$$

Para cada muestra se extrae un patch $P_l$. Primero se calcula un centroide de energía para obtener una alineación inicial. Si $P_l[a,b]$ es el valor del patch en frecuencia discreta $a$ y tiempo discreto $b$, y $w_{a,b}^{(l)}$ es un peso de saliencia, entonces:

$$c_t^{(l)} = \frac{\sum_{a,b} b w_{a,b}^{(l)}}{\sum_{a,b} w_{a,b}^{(l)} + \varepsilon}$$

$$c_f^{(l)} = \frac{\sum_{a,b} a w_{a,b}^{(l)}}{\sum_{a,b} w_{a,b}^{(l)} + \varepsilon}$$

Luego se refina la alineación con una búsqueda local de desplazamientos. La aplicación escoge una muestra de referencia $P_{ref}$ mediante un medoide inicial y, para cada muestra, busca:

$$\Delta_l^{*} = \arg\max_{\Delta t, \Delta f} \operatorname{Sim}_w(P_{ref}, \operatorname{shift}(P_l, \Delta t, \Delta f))$$

La función $\operatorname{Sim}_w$ es una correlación ponderada. Si $A$ y $B$ son dos patches normalizados, y $w_j$ es el peso saliente del píxel $j$, se define:

$$\bar{A}_w = \frac{\sum_j w_j A_j}{\sum_j w_j + \varepsilon}, \qquad \bar{B}_w = \frac{\sum_j w_j B_j}{\sum_j w_j + \varepsilon}$$

$$\operatorname{Sim}_w(A,B) = \frac{\sum_j w_j (A_j - \bar{A}_w)(B_j - \bar{B}_w)}{\sqrt{\sum_j w_j (A_j - \bar{A}_w)^2}\sqrt{\sum_j w_j (B_j - \bar{B}_w)^2} + \varepsilon}$$

Esta ponderación busca alinear la huella espectral del canto, no el fondo acústico.

Una vez alineadas las muestras se obtienen patches $P_1^{aligned}, \dots, P_L^{aligned}$. El selector Método de plantilla construye la plantilla efectiva $P^{*}$ con una de estas opciones:

Medoide: usa la muestra real más representativa del conjunto:

$$P^{*} = P_k^{aligned}, \qquad k = \arg\max_i \frac{1}{L-1}\sum_{j \neq i} \operatorname{Sim}(P_i^{aligned}, P_j^{aligned})$$

Mediana alineada: combina las muestras alineadas por mediana píxel a píxel:

$$P^{*}[a,b] = \operatorname{mediana}\{P_1^{aligned}[a,b], \dots, P_L^{aligned}[a,b]\}$$

Promedio alineado: promedia las muestras alineadas:

$$P^{*}[a,b] = \frac{1}{L}\sum_{l=1}^{L} P_l^{aligned}[a,b]$$

Consenso NCC: usa la alineación por similitud ponderada y conserva una huella robusta mediante un percentil alto por píxel:

$$P^{*}[a,b] = Q_{0.70}(P_1^{aligned}[a,b], \dots, P_L^{aligned}[a,b])$$

Consenso ponderado: promedia las muestras alineadas con pesos de calidad $\alpha_l$ derivados de la similitud de alineación:

$$P^{*}[a,b] = \frac{\sum_{l=1}^{L} \alpha_l P_l^{aligned}[a,b]}{\sum_{l=1}^{L} \alpha_l + \varepsilon}$$

En todos los casos la vista previa muestra la plantilla efectiva $P^{*}$ que se usará para construir el embedding o la representación de comparación.

3. Embedding del patch

Cada patch se normaliza localmente para reducir el efecto de intensidad absoluta:

$$P_{\text{norm}} = \frac{P - \min(P)}{\max(P) - \min(P) + \varepsilon}$$

Luego se redimensiona a $48 \times 48$ y se aplana:

$$z = \operatorname{vec}(\operatorname{resize}(P_{\text{norm}}, 48, 48))$$

Como $48 \times 48 = 2304$, cada patch queda representado inicialmente por un vector $z \in \mathbb{R}^{2304}$.

Después se estandariza y se normaliza con norma euclidiana:

$$\tilde{z} = \frac{z - \mu_z}{\sigma_z + \varepsilon}$$

$$e = \frac{\tilde{z}}{\|\tilde{z}\|_2 + \varepsilon}$$

El vector $e$ es el embedding usado para comparar la plantilla con otras ventanas del audio.

4. Ventanas candidatas y scores

La aplicación recorre el audio con ventanas candidatas del mismo tamaño temporal y frecuencial que la ROI. Si la separación entre ventanas es $\Delta t$, se evalúan ventanas en tiempos:

$$t_j = j\Delta t$$

Para cada ventana candidata $C_j$ se extrae un patch $P_j$, se calcula su embedding $e_j$ y se obtiene un score de similitud:

$$s_j = \operatorname{score}(R, C_j)$$

La app mantiene todos los métodos de comparación en una escala común de 0 a 1, donde 1 indica mayor parecido.

Para coseno y correlación normalizada se usa:

$$s_j = \max(0, e_R^\top e_j)$$

Para distancia euclidiana normalizada:

$$d_j = \|e_R - e_j\|_2$$

$$s_j = \max\left(0, 1 - \frac{d_j}{2}\right)$$

Como los embeddings están normalizados, esta transformación permite interpretar también la métrica euclidiana como un score entre 0 y 1.

Los métodos de correlación cruzada comparan representaciones reducidas de la plantilla y la ventana candidata mediante correlación normalizada local. En la correlación cruzada temporal se permite un pequeño desplazamiento horizontal. En la correlación cruzada 2D se permiten desplazamientos en tiempo y frecuencia:

$$s(P,Q) = \max_{\Delta t, \Delta f} \operatorname{NCC}(P, \operatorname{shift}(Q, \Delta t, \Delta f))\, \operatorname{pen}(\Delta t, \Delta f)$$

La correlación 2D multi-escala agrega pequeñas escalas globales de tiempo y frecuencia:

$$s(P,Q) = \max_{a_t, a_f, \Delta t, \Delta f} \operatorname{NCC}(P, T_{a_t,a_f,\Delta t,\Delta f}(Q))\, \operatorname{pen}(a_t, a_f, \Delta t, \Delta f)$$

Aquí $a_t$ y $a_f$ son escalas temporales y frecuenciales cercanas a 1. La penalización evita que deformaciones o desplazamientos grandes produzcan coincidencias demasiado permisivas. Este método es más flexible, pero puede tardar más.

5. De ventanas a eventos

Las ventanas evaluadas no son todavía eventos finales. Muchas ventanas consecutivas pueden representar el mismo sonido. Por eso se agrupan candidatos cercanos y se conserva solo el mejor score de cada grupo temporal.

Si $w_R$ es el ancho de la plantilla en frames, se define una separación mínima interna entre eventos:

$$\Delta_{\text{evento}} = \gamma w_R$$

donde $\gamma$ es una constante interna. En la práctica esto evita que un solo evento acústico genere muchas cajas repetidas.

Después de esta supresión temporal, se obtiene una lista de picos o islas:

$$P = \{p_1, p_2, \dots, p_M\}$$

donde cada $p_i$ tiene un score $s_i$ y representa un candidato temporalmente separado.

6. Autoajuste de parámetros

Cuando está activo Autoajustar parámetros al buscar, la app estima automáticamente dos valores: la separación entre ventanas $\Delta t$ y el score mínimo $\tau$.

La separación entre ventanas se propone a partir de la duración de la ROI:

$$\Delta t = \operatorname{clip}(\alpha T_R, \Delta t_{\min}, \Delta t_{\max})$$

donde $\alpha$ es una fracción interna y $\operatorname{clip}$ limita el valor para que no sea ni demasiado fino ni demasiado grande.

Para estimar el score mínimo, primero se excluye la zona de la plantilla original al calcular el corte automático, de modo que el match trivial de la propia ROI no domine el umbral. Luego se ordenan los scores de los picos:

$$s_{(1)} \geq s_{(2)} \geq \dots \geq s_{(M)}$$

El umbral automático combina varias señales:

  • un umbral de codo $\tau_{\text{codo}}$, obtenido de la curva de scores ordenados;
  • un umbral robusto de fondo $\tau_{\text{ruido}}$, basado en mediana y dispersión robusta;
  • un umbral relativo $\tau_{\text{rel}}$, definido como una fracción del mejor score;
  • barandas internas para evitar dos extremos: cientos de cajas o una sola caja si hay más candidatos razonables.

Conceptualmente:

$$\tau_0 = \max(\tau_{\text{codo}}, \tau_{\text{ruido}}, \tau_{\text{rel}}, \tau_{\min})$$

Luego se ajusta $\tau_0$ usando límites internos de cantidad:

$$N_{\min}^{\text{objetivo}} \leq N(\tau) \leq N_{\max}^{\text{auto}}$$

donde $N(\tau)$ es el número de picos con score mayor o igual a $\tau$. Estos límites son objetivos, no reglas para inventar resultados. Si no existen suficientes coincidencias razonables, la app muestra las que existan.

Finalmente se conservan los eventos que cumplen:

$$s_i \geq \tau$$

Cuando termina una búsqueda con autoajuste, la casilla de autoajuste se desmarca automáticamente. Así los ajustes manuales posteriores de score y separación sí tienen efecto inmediato. Al subir un nuevo audio, la casilla vuelve a quedar activa.

7. Interpretación

Las cajas azules son candidatos similares a la plantilla, no identificaciones taxonómicas definitivas. El flujo recomendado es usar la búsqueda automática como primer filtro, revisar visualmente los candidatos y exportar los resultados para validación posterior.

Exportación

Los resultados pueden descargarse como CSV, XLSX o TXT compatible con etiquetas espectrales de Audacity. El nombre base de exportación coincide con el nombre del audio subido.

audio,tmin,tmax,fmin,fmax,etiqueta,score,estado

El TXT Audacity usa dos líneas por coincidencia: una para tiempos y etiqueta, y otra para la banda de frecuencia. Estos resultados son candidatos, no validaciones definitivas.

Problemas frecuentes

El HTML abre, pero no funciona el audio o el worker

Abre el proyecto desde http://localhost:8000, no con doble clic.

El espectrograma tarda

Audios largos pueden tardar porque el cálculo se hace localmente en el navegador. Espera a que termine el indicador de procesamiento.

Demasiadas cajas azules

Sube el score mínimo o marca una ROI más específica.

Muy pocas cajas

Baja el score mínimo, aumenta la banda o prueba una ROI más representativa.

Proyecto web estático. Puede publicarse en GitHub Pages. El audio se procesa localmente en el navegador del usuario.

Modo multi-plantilla

Desde la versión v16, la aplicación permite trabajar con varias plantillas acústicas en el mismo audio. Cada plantilla tiene su propia etiqueta, color, coordenadas, parámetros de búsqueda y resultados asociados.

Si no se escribe una etiqueta, la aplicación asigna nombres automáticos como fonotipo1, fonotipo2 y así sucesivamente. Los chips horizontales permiten saltar entre plantillas, mientras que el carrusel permite avanzar o retroceder una plantilla a la vez.

La primera vez que se pulsa Buscar similares, se procesan todas las plantillas guardadas. Después de esa primera corrida, las búsquedas siguientes se aplican solo a la plantilla activa, para permitir ajustes finos. El botón Limpiar también actúa localmente sobre la plantilla activa.

El casillero Ver cajas permite mostrar u ocultar las cajas inferidas de la plantilla activa sin borrar los resultados ni afectar las exportaciones.

En la tabla de resultados, la columna de etiqueta es editable. Si se cambia la etiqueta de una coincidencia, el cambio se propaga a todas las coincidencias de la misma plantilla y al chip correspondiente.

Flujo multi-plantilla simplificado

Desde la versión v17, el etiquetador trabaja como un gestor de plantillas acústicas. Cada plantilla conserva su etiqueta, color, coordenadas, parámetros de búsqueda y coincidencias.

El panel Plantilla permite navegar con chips superiores y carrusel inferior. El botón Agregar plantilla + guarda o actualiza la plantilla actual y abre una nueva plantilla para seguir marcando. El botón Quitar plantilla − elimina la plantilla activa.

El botón Buscar coincidencias guarda automáticamente la plantilla marcada si la caja es válida y busca todas las plantillas válidas, aplicando el autoajuste por plantilla. Luego el panel Búsqueda permite revisar y ajustar cada plantilla con la misma navegación por chips y carrusel.

La tabla de resultados muestra primero el número de fila (#) y luego la etiqueta en una cápsula coloreada editable. Al editar esa etiqueta, se actualiza la plantilla completa, sus coincidencias y las exportaciones CSV, XLSX y TXT Audacity.

Caché por método de plantilla

Desde la v37, cuando se activa Usar varias muestras, la vista previa de cada método de plantilla se conserva por separado. Si las muestras no cambian, volver a un método ya calculado reutiliza la caché y no recalcula la plantilla compuesta.

La caché se invalida solamente cuando cambian las muestras, se carga otro audio o se reconstruye el espectrograma. Cambiar entre métodos de plantilla ya calculados debe ser inmediato.

Correlación cruzada local

La versión v39 agrega un método de comparación llamado correlación cruzada. A diferencia de la correlación normalizada simple, que compara directamente la plantilla contra una ventana ya alineada, la correlación cruzada prueba pequeños desplazamientos internos en tiempo y frecuencia y conserva el mejor score.

Si $P$ es la plantilla y $Q$ una ventana candidata, la idea es:

$$s(P,Q)=\max_{\Delta t,\Delta f} \operatorname{NCC}(P, Q_{\Delta t,\Delta f}) \cdot \exp(-\lambda(|\Delta t|+|\Delta f|))$$

La función $\operatorname{NCC}$ es una correlación normalizada local. El factor exponencial penaliza desplazamientos grandes para evitar que el método se vuelva demasiado permisivo. El score final se mantiene entre 0 y 1 para conservar el mismo flujo de autoajuste.

Este método puede detectar coincidencias ligeramente desalineadas respecto a la plantilla, aunque normalmente tarda más que coseno, correlación normalizada o euclidiana.

Actualización v41: correlación cruzada 2D

La versión v41 retira coseno cruzado y agrega correlación cruzada 2D. La opción correlación cruzada conserva una tolerancia local temporal; la nueva opción correlación cruzada 2D permite tolerancias pequeñas tanto en tiempo como en frecuencia.

Para una plantilla reducida P y una ventana candidata reducida Q, se evalúan desplazamientos pequeños Δt y Δf:

score = max NCC(P, shift(Q, Δt, Δf)) × penalización(Δt, Δf)

La penalización evita que desplazamientos grandes produzcan coincidencias demasiado permisivas. El resultado final sigue en la escala 0–1, igual que los demás métodos de comparación, para mantener compatible el autoajuste.

Actualización v42: correlación 2D multi-escala

La versión v42 agrega el método correlación 2D multi-escala. Este método extiende la correlación cruzada 2D: además de pequeños desplazamientos temporales y frecuenciales, prueba pequeñas variaciones globales de duración y escala frecuencial. No es un warping libre tiempo-frecuencia; es una aproximación controlada, con pocas escalas y penalización por deformación.

score = max NCC(P, transform(Q, escala_t, escala_f, Δt, Δf)) × penalización

Exportación de resultados

Los archivos CSV, XLSX y TXT Audacity se descargan usando el mismo nombre base del audio original, cambiando solo la extensión. Si el audio se llama audio.mp3, las exportaciones serán audio.csv, audio.xlsx y audio.txt.

Actualización v38: caché persistente por método

La caché de plantillas compuestas se guarda ahora dentro de cada plantilla y se organiza por método. Así, si ya se calculó consenso NCC, consenso ponderado, medoide, mediana alineada o promedio alineado, volver a ese método no recalcula la plantilla mientras las muestras no hayan cambiado.

También se evita que el soporte compuesto mostrado en los campos se agregue accidentalmente como una muestra nueva al ejecutar la búsqueda. Esto mantiene estable la firma de muestras y evita perder la caché. Además, la app precalienta la caché interna del worker de búsqueda cuando se construye una vista previa compuesta.

Optimización y caché de plantillas compuestas

Cuando se activa Usar varias muestras, la construcción de la plantilla compuesta puede requerir alinear varias muestras y calcular un consenso. Para evitar que la interfaz recalcule el mismo resultado una y otra vez, la app conserva en caché la plantilla compuesta asociada a la combinación de muestras, método de plantilla y configuración visual del espectrograma.

Si no cambian las muestras ni la configuración del espectrograma, cada método ya calculado permanece en caché. Cambiar de un método a otro no invalida la caché; al volver a un método previamente calculado, la vista previa debe mostrarse de inmediato. El caché solo se invalida al agregar/quitar muestras, cargar otro audio o reconstruir el espectrograma. El panel de muestras muestra una barra de progreso compacta durante la primera construcción de cada método.

Caché de plantillas compuestas

La aplicación conserva en memoria cada plantilla compuesta calculada para una combinación fija de muestras, método de plantilla y configuración visual. Si el usuario vuelve a un método ya calculado, la vista previa y la búsqueda reutilizan el resultado en caché. Esto evita reconstruir el consenso al cambiar entre métodos.

El caché se invalida únicamente cuando cambian las muestras, el audio o la configuración del espectrograma. En particular, cambiar de consenso NCC a medoide y volver a consenso NCC no debería recalcular si las muestras no cambiaron.

El medoide usa una ruta rápida basada en embeddings 48×48 para escoger una muestra real representativa. Los métodos de consenso siguen usando alineación, pero su resultado queda guardado.
